initialize 

Procedure initialize

Initialization procedure Made public to allow for any session to reset the PIT.
As the actual status is persisted in a global context, initializing PIT will re-initialize all PIT packages

set_context 

Procedure set_context(p_log_level    in integer,
                      p_trace_level  in integer,
                      p_trace_timing in boolean,
                      p_module_list  in varchar2,
                      p_focus        in varchar2 default 'ACTIVE')

Procedure to change the settings in the global PIT_CONTEXT

Parameters
p_log_level   New log level.
p_trace_level   New trace level
p_trace_timing   Flag to switch timing on or off
p_module_list   Colon-separated list of output modules used
p_focus   Indicates whether changes are made for the active session only (ACTIVE) or as a new default setting (DEFAULT). Defaults to ACTIVE $usage This procedure is used when log settings shall be changed dynamically Normal usage is to overwrite log settings as defined in the parameters for a given session.

reset_active_context 

Procedure reset_active_context

Procedure to reset log settings to the default settings

Usage
If settings for a session were changed, calling this procedure resets these settings to the default settings as defined by the parameters

reset_context 

Procedure reset_context

Procedure to reset the complete context to default settings

Usage
Use this procedure if all session settings shall be reset to the default settings. Be aware that calling this procedure will reset ALL log settings of ALL sessions to default

log 

Procedure log(p_level        in integer,
              p_message_name in varchar2,
              p_arg_list     in msg_args)

Logs messages

Usage
This procedure is called from PIT. It takes the message_name and constructs an instance of MESSAGE_TYPE for it. It then calls any log procedure of all active output modules and passes the message.

Parameters
p_level   Log-level. Allows to decide whether PIT should log or not.
p_message_name   Name of the message to log
p_arg_list   List of replacement values for the message

enter 

Procedure enter(p_action      in varchar2,
                p_module      in varchar2,
                p_params      in msg_params,
                p_trace_level in number)

Traces entering a procedure

Usage
This procedure is called from PIT to trace entering a procedure. It will determine timing information and maintain a call stack.

Parameters
p_action   Nam of the block that is executed. Normally the procedure or function name within a package.
p_module   Unit that contains the method that is executed. Either the package name or ANONYMOUS, PROCEDURE, FUNCTION or TRIGGER
p_params   List of parameters which is passed into tracing. Only varhar2 is allowed as a parameter value with a maximum of 4000 byte.
p_trace_level   Trace level to allow for selective tracing.

p_trace_level   Trace level to allow for selective tracing.

raise_error 

Procedure raise_error(p_message_name varchar2, p_arg_list in msg_args)

Raises an error

Usage
This procedure is called from PIT. It takes the message_name and constructs an instance of MESSAGE_TYPE for it. It then calls raises an error with the respective message that can be caught by the exception block. Messages with severity error or fatal have an associated error called <MESSAGE_NAME>_ERR that can be used to capture the error.

Parameters
p_message_name   Name of the message to log
p_arg_list   List of replacement values for the message
